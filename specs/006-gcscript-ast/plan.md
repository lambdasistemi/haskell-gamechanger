# Implementation Plan: GameChanger.GCScript — upstream AST + JSON round-trip

**Branch**: `006-gcscript-ast` | **Date**: 2026-04-20 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-gcscript-ast/spec.md`

## Summary

Ship `GameChanger.GCScript`: a Haskell AST that round-trips the real
upstream GameChanger wallet DSL. Structured constructors for the
twelve function kinds we depend on (`buildTx`, `signTxs`,
`submitTxs`, `buildFsTxs`, `signDataWithAddress`,
`verifySignatureWithAddress`, `query`, `plutusScript`, `plutusData`,
`nativeScript`, `macro`, recursive `script`); `Unsupported` fallback
preserving `Aeson.Value` bodies for everything else. Polymorphic
`run` covering object / array / ISL-string shapes. `CommonAttrs`
for the shared optional fields every call carries (`title`,
`description`, `exportAs`, `args`, `argsByKey`, `returnURLPattern`,
`require`, `return`). Round-trip proven on a curated 15-file
corpus from `GameChangerFinance/gamechanger.wallet/examples`, pinned
by commit sha.

## Technical Context

**Language/Version**: Haskell 2010 on GHC 9.6+ (inherited from
`haskell-gamechanger.cabal`).
**Primary Dependencies**: `aeson ^>=2.2`, `aeson-pretty` (for
deterministic emit if needed), `containers` (`Data.Map.Strict`),
`text`, `bytestring`, `vector` (for array-shaped `run`).
**Storage**: N/A (pure AST + codec).
**Testing**: `tasty`, `tasty-hunit`, `tasty-golden`,
`tasty-quickcheck` (for generator-based round-trip).
**Target Platform**: native GHC (no WASM, per constitution §8).
**Project Type**: library (additive new module + tests).
**Performance Goals**: no hard perf target — correctness and
determinism come first.
**Constraints**: pure (no `IO` in the AST module); byte-level
deterministic emit; no `cardano-api` dependency (§1).
**Scale/Scope**: the 43-function upstream universe, of which we
structure twelve; the rest fall through `Unsupported`.

## Complexity Tracking

with Aeson codec. The only non-trivial decision is the
polymorphic `RunBlock`, justified in `research.md` R2.
